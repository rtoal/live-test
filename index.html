Practice TDD

Begin by writing a test, using Chai for assertions. You’ll see a red bar on the left of the test because it is (or should be) failing. Next, write some code to make the test pass and get that green bar.

Tests and coverage are live (run continuously). Navigate between the test panel and editor with Ctrl+Enter. Navigate among the tests with Tab and Shift+Tab.

You can add a new test by hitting Shift+Enter in a test box.

Each test is independent. If you need some code to run before each test (e.g., data structure initialization), enter it in the Setup box.

If the first line of a test is a comment, it will be used in the exported test script as the test’s description.
